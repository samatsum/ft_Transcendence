# USER_DOC — プレイヤー／評価者向けマニュアル

このドキュメントは、cub3D の遊び方と、自作マップ（`.cub` ファイル）を作成するためのルールをまとめたものです。**実装と一致する内容のみ** を記載しています。

---

## 1. 起動

```
make
./cub3D maps/fps_map/<map.cub>      # FPS モード
./cub3D maps/rsp_map/<map.cub>      # RSP モード（じゃんけん鬼ごっこ）
```

第 2 引数は指定しません。`maps/fps_map/` 配下のマップは FPS モード、`maps/rsp_map/` 配下のマップは RSP モードとして自動判定されます。

```
./cub3D maps/fps_map/1.cub             # FPS
./cub3D maps/fps_map/enemy_line.cub    # FPS（敵の挙動確認用）
./cub3D maps/fps_map/door_test.cub     # FPS（扉の確認用）
./cub3D maps/rsp_map/rsp.cub           # RSP（じゃんけん鬼ごっこ）
```

## 2. 操作

| キー | 動作 |
|---|---|
| `W` / `S` | 前進 / 後退 |
| `A` / `D` | 左右への平行移動（ストレイフ） |
| `←` / `→` | 視点を左右に回転 |
| `1` | ピストルを装備（**FPS モード専用**） |
| `2` | フラッシュライト（懐中電灯）を装備（**FPS モード専用**） |
| `3` | 素手（移動が 1.5 倍速くなる。**FPS モード専用**） |
| `Space` | 射撃（ピストル装備時のみ。クールダウンあり。**FPS モード専用**） |
| `I` | UI（ミニマップ・収集進捗）の表示切替 |
| `O` | クロスヘアの表示切替 |
| `L` | 距離による陰影（シェード）の切替 |
| `Esc` / ウィンドウの × | 終了 |

> 移動は WASD、回転は左右矢印のみです（上下矢印・Q・E は未割り当て）。**RSP モードでは `1` / `2` / `3` / `Space` は無効** で、武器切り替え・射撃はできません（じゃんけんは接触時に自動判定。詳細は §5）。

## 3. ゲーム要素（FPS モード）

- **収集アイテム**（マップ文字 `k`〜`o`、`.cub` の `OC*` テクスチャ）に触れると獲得され、UI に進捗（`Collect: x / y`）が表示されます。すべて集めると `GG !` の表示になります。収集対象が 1 つもないマップでは `Nothing to collect !` と表示されます。
- **扉**（マップ文字 `D`）は、**収集アイテムをすべて集めるまでは壁** として振る舞い、集め終わると床に変わって **開放** されます。収集対象が無いマップでは扉は閉じたままです。
- **敵**（マップ文字 `M`）はプレイヤーの方角に応じた 8 方向スプライトで描画されます。普段は巡回し、**正面の一定範囲（視界）にプレイヤーが入り、間に遮蔽物（壁・通行不可オブジェクト・閉じた扉）がない** と検知して追跡へ切り替わります。見失っても一定時間（既定 5 秒、`.cub` の `ET` で変更可）は追跡を続け、時間切れで巡回へ戻ります。ピストルで撃って既定 5 ヒット（`.cub` の `EH` で変更可）させると倒せます。移動速度は `.cub` の `ES` で調整できます（プレイヤーの移動速度とは独立）。
- **巡回路**（マップ文字 `P`）は敵が巡回するための通路です。`P` セルをループ状に並べておくと、敵はその上を周回します。敵が `P` 上にいないときは、最も近い `P` へ自動で復帰します。`P` を 1 つも置かなければ、敵は検知時のみ動き、それ以外は待機します。
- **通行不可オブジェクト**（`a`〜`e`、`OI*`）は壁と同様に進行を妨げます。
- **通行可オブジェクト**（`f`〜`j`、`OP*`）は装飾として描画されますが、通り抜けられます。周囲をわずかに明るくするスポットライト効果があります。

## 4. `.cub` ファイルの記述ルール

### 4.1 設定ヘッダ

ファイル前半に、解像度・テクスチャ・色・各種パラメータを記述します。同じキーは **1 度だけ** 記述できます。記述順は自由ですが、**マップ本体より前** に置く必要があります。

```
R 1920 1080                                # 解像度 (幅 高さ)

NO ./textures/wall/Wall_1.xpm              # 北向きの壁テクスチャ
SO ./textures/wall/Wall_2.xpm              # 南向きの壁テクスチャ
WE ./textures/wall/Wall_3.xpm              # 西向きの壁テクスチャ
EA ./textures/wall/Wall_4.xpm              # 東向きの壁テクスチャ

ST ./textures/wall/Wall_mossy.xpm          # 天井テクスチャ (任意)
FT ./textures/wall/Wall_tile.xpm           # 床テクスチャ (任意)

F 184,113,39                               # 床の色 R,G,B (テクスチャ未指定時のフォールバック)
C 51,198,227                               # 天井の色 R,G,B (テクスチャ未指定時のフォールバック)

OI1 ./textures/object/Object_barrel.xpm    # 通行不可オブジェクト 1種目 (マップ文字 'a')
OP1 ./textures/object/Object_Green_Light.xpm  # 通行可オブジェクト 1種目 (マップ文字 'f')
OC1 ./textures/object/Object_star.xpm      # 収集アイテム         1種目 (マップ文字 'k')

MS 0.05                                     # 移動速度     (任意, 0 より大。既定 0.05)
RS 0.05                                     # 回転速度     (任意, 0 より大。既定 0.05)
FOV 0.66                                    # 視野角       (任意, 大きいほど広角。既定 0.66)
ET 5.0                                      # 敵の追跡秒数 (任意, 見失うまでの秒数。既定 5.0)
ES 0.1                                      # 敵の移動速度 (任意, 0 より大。既定 0.1, 移動速度と独立)
EH 5                                        # 敵のHP       (任意, 0 より大の整数。撃破に要するヒット数。既定 5)
```

サポートされる画像形式は `.xpm` のみです。

#### 設定キー一覧

| キー | 内容 | 必須 |
|---|---|---|
| `R` | 解像度（幅 高さ） | 必須 |
| `NO` / `SO` / `WE` / `EA` | 各方角の壁テクスチャ | 必須 |
| `ST` | 天井テクスチャ | 任意（未指定時は `C` の色） |
| `FT` | 床テクスチャ | 任意（未指定時は `F` の色） |
| `F` / `C` | 床 / 天井のフォールバック色 `R,G,B` | テクスチャ未指定時に使用 |
| `OI1`〜`OI5` | 通行不可オブジェクトのテクスチャ（最大 5 種） | 任意 |
| `OP1`〜`OP5` | 通行可オブジェクトのテクスチャ（最大 5 種） | 任意 |
| `OC1`〜`OC5` | 収集アイテムのテクスチャ（最大 5 種） | 任意 |
| `MS` / `RS` / `FOV` | 移動速度 / 回転速度 / 視野角 | 任意（既定値で動作） |
| `ET` / `ES` / `EH` | 敵の追跡秒数 / 移動速度 / HP | 任意（既定値で動作） |

> **注:** オブジェクトのキーは **番号付き** の `OI1`〜`OI5` / `OP1`〜`OP5` / `OC1`〜`OC5` を使用します（番号なしの裸キー `OI` / `OP` / `OC` はサポートされていません）。

### 4.2 マップ本体

ヘッダの後にマップを記述します。マップは `.cub` の **最後** に置く必要があります。各セルはスペースで区切ります（例: `1 0 0 1`）。

使用可能な文字は以下のとおりです。

| 文字 | 意味 | 対応テクスチャ |
|---|---|---|
| `0` | 通行可能な床 | — |
| `1` | 壁（通行不可） | `NO`/`SO`/`WE`/`EA` |
| `a`〜`e` | 通行不可オブジェクト（5 種） | `OI1`〜`OI5` |
| `f`〜`j` | 通行可オブジェクト（5 種, 通り抜け可） | `OP1`〜`OP5` |
| `k`〜`o` | 収集アイテム（5 種, 触れると獲得） | `OC1`〜`OC5` |
| `D` | 扉（収集完了で床に変わり開放。それまでは壁） | `NO`/`SO`/`WE`/`EA` で描画 |
| `M` | 敵の出現地点 | 敵テクスチャ（8 方向, 内蔵） |
| `P` | 巡回路（敵が周回する通路。床として通行可能） | —（描画は床扱い） |
| `N` / `S` / `E` / `W` | プレイヤー初期位置と向き（RSP ではチームスポーン。§5） | — |
| 空白 | 描画されない領域 | — |

オブジェクトの文字とテクスチャ番号の対応:

- 通行不可: `a`→`OI1`, `b`→`OI2`, `c`→`OI3`, `d`→`OI4`, `e`→`OI5`
- 通行可　: `f`→`OP1`, `g`→`OP2`, `h`→`OP3`, `i`→`OP4`, `j`→`OP5`
- 収集　　: `k`→`OC1`, `l`→`OC2`, `m`→`OC3`, `n`→`OC4`, `o`→`OC5`

ルール:

1. プレイヤーのスタート地点 `N` / `S` / `E` / `W` は、**FPS モードではちょうど 1 つ** 必要です（RSP モードの要件は §5）。
2. マップは壁 `1` で囲まれている必要があります。隙間や開いた境界はパースエラーになります。
3. 字句解析後、すべての行の有効列数（空白を除く）が一致している必要があります。
4. 使用したオブジェクト文字に対応するテクスチャキー（例: `b` を使うなら `OI2`）を定義してください。
5. 使用できる文字は上表のもののみです（`2` / `3` / `4` などはサポートされません）。

> **巡回路の設計ヒント:** `P` セルを **閉じたループ状** に並べると、敵が安定して周回します。敵（`M`）は `P` の上、または `P` に隣接させて配置すると、巡回にスムーズに乗ります。分岐や行き止まりのある `P` でも動作しますが、ループ状が最も意図どおりになります。`P` を置かないマップでは、敵は検知時のみプレイヤーを追跡し、それ以外は待機します。

### 4.3 解像度の制約

実行時、要求された解像度は内部で次の範囲にクリップされます。

- 最小: 848 × 480
- 最大: 1920 × 1080

これより大きな値を書いても、自動的に 1920 × 1080 まで縮められます。アスペクト比に応じて FOV が内部補正されます。

## 5. RSP モード（じゃんけん鬼ごっこ）

`./cub3D maps/rsp_map/<map.cub>` で起動する対戦的なモードです。プレイヤーと NPC が **赤チーム** と **青チーム** に分かれ、接触したときの **じゃんけん** で勝敗を決めます。

### ルール

- **チームとスポーン**: `N` / `W` のスポーンが **赤チーム**、`S` / `E` のスポーンが **青チーム** です。起動時に各チームから 2 地点ずつ（計 4 地点）が選ばれ、そのうち 1 つがプレイヤー、残り 3 つが NPC になります。
- **手**: 各戦闘員はグー / チョキ / パーのいずれかを持ちます。手は頭上のスプライトで表示されます（赤／青のハンド画像）。
- **勝敗**: 異なるチームの戦闘員どうしが接触すると、その場でじゃんけん判定が行われ、**負けた側が即リスポーン** します（勝ち・あいこ・味方どうしは何も起きません）。
- **手の変化**: 自陣のスポーンマス（赤なら `N`/`W`、青なら `S`/`E`）に **新たに踏み込んだ瞬間** に、手が今と違うものへランダムに変わります。リスポーン時も手が変わります。
- **NPC の AI**: 最寄りの異チーム戦闘員を見て、**勝てる手なら追跡・負ける手なら逃走・あいこや相手不在なら徘徊** します。NPC の移動速度はプレイヤーが追える／逃げられるよう抑えられています。
- **操作**: プレイヤーのゲーム内操作は移動のみです（`1` / `2` / `3` の武器切り替えと射撃はできません）。UI表示切替・終了操作は共通です。

### RSP マップの作り方

- 赤チーム用の `N` または `W` のスポーンを **2 地点以上**、青チーム用の `S` または `E` のスポーンを **2 地点以上** 配置します（各チーム 2 地点に満たないと戦闘員を生成できず起動に失敗します）。
- スポーンマスには任意でマーカー（スポットライト）が描画されます。通行可オブジェクトの `OP2` に赤チーム用、`OP3` に青チーム用のテクスチャを指定すると、各スポーンにその色のマーカーが表示されます（`maps/rsp_map/rsp.cub` を参照）。
- ハンド画像（`textures/hand/Hand_<Red|Blue>_<Rock|Scissors|Paper>.xpm`）はエンジンが内蔵で読み込むため、`.cub` での指定は不要です。

## 6. うまく動かないとき

- **`Error: map must be in maps/fps_map/ or maps/rsp_map/.`** … マップは `maps/fps_map/` または `maps/rsp_map/` 配下に置いてください。
- **`Error: too many arguments.`** … 引数は `./cub3D <map.cub>` の 1 つだけ指定してください。第 2 引数の `FPS` / `RSP` は使いません。
- **`Error: map file must end with .cub.`** … マップファイルの拡張子が `.cub` ではありません。
- **`Error: failed to open map file.`** … 指定したマップファイルが存在しない、または読み取り権限がありません。
- **`Error: invalid map content. check syntax, walls, characters, and spawns.`** … `.cub` の構文、壁閉じ、使用文字、スポーン指定のいずれかが不正です。
- **`Error: failed to create RSP combatants. need 2 red spawns (N/W) and 2 blue spawns (S/E).`** … RSP マップには赤 `N`/`W` を 2 地点以上、青 `S`/`E` を 2 地点以上置いてください。
- **`Error: failed to load map texture(s). check the texture path printed above.`** … `.cub` のテクスチャパスが読めません。直前に表示される `Failed to load texture: ...` のパスを確認してください。
- **画面が真っ黒** … テクスチャパスが間違っている、または XPM 以外の画像形式を指定している可能性があります。
- **敵が巡回しない／その場で止まる** … マップに `P`（巡回路）が無い、または敵が `P` から離れすぎている可能性があります。`P` をループ状に配置し、敵をその近くに置いてください。

## 7. 結果スクリーンショット

結果画面は自動で BMP 保存されます。FPS は `./screenshot/fps_screen/fps_YYYYMMDD_HHMMSS_usec.bmp`、RSP は `./screenshot/rsp_screen/rsp_YYYYMMDD_HHMMSS_usec.bmp` に出力します。手動保存用のキーバインドやコマンドラインオプションはありません。
